Kryptografische Architektur

Wer weiß was?

Auf unserem Server werden nur pseudonymisierte Namen der Schüler:innen gespeichert. Das heißt der Server kennt das Beziehungsnetzwerk allerdings nicht mit Klarnamen.

Die Lehrkraft erhält am Ende eine Aufteilung der Klasse in zwei Hälften. Das heißt die Lehrkraft kennt die Klarnamen der Schüler:innen und diese Aufteilung die durch das Beziehungsnetzwerk informiert ist. Allerdings kennt die Lehrkraft nicht welche Beziehungen welche:r Schüler:in eingetragen hat.

Technische Umsetzung

Sobald eine neue Klasse erstellt wird werden im Browser der Lehrkraft ein Klassenpasswort und ein public/private key pair zufällig generiert. Der PrivateKey wird mit dem Klassenpasswort symmetrisch verschlüsselt und mit dem PublicKey zusammen an den Server übertragen. Das Klassenpasswort ist nur der Lehrkraft bekannt und wird nie an den Server übertragen.

In der Schüler:innen-Ansicht werden alle eingetragenen Namen mit dem public key verschlüsselt. Zusätzlich wird ein hash aus jedem Namen erstellt der im Server als Pseudonym verwendet wird. Da dies im Browser der Schüler:innen passiert werden nie Klarnamen übertragen oder gespeichert.

Der Server erstellt die Klasseneinteilung anhand der Pseudonyme (hash des Klarnamens). Als Ergebnis wird dann die Liste von (mit dem PublicKey) verschlüsselten Namen an die Lehrkraft übertragen. Zusätzlich wird auch der symmetrisch verschlüsselte PrivateKey übertragen. Ausschließlich die Lehrkraft kennt das Klassenpasswort mit dem sie den PrivateKey entschlüsseln kann. Mit diesem werden dann die Namen der Schüler:innen entschlüsselt.

Server-Log-Files

Der Provider der Seiten erhebt und speichert automatisch Informationen in so genannten Server-Log Files, die Ihr Browser automatisch an uns übermittelt. Dies sind:

  • Browsertyp und Browserversion
  • verwendetes Betriebssystem
  • Referrer URL
  • Hostname des zugreifenden Rechners
  • Uhrzeit der Serveranfrage

Diese Daten sind nicht bestimmten Personen zuordenbar. Eine Zusammenführung dieser Daten mit anderen Datenquellen wird nicht vorgenommen. Wir behalten uns vor, diese Daten nachträglich zu prüfen, wenn uns konkrete Anhaltspunkte für eine rechtswidrige Nutzung bekannt werden.
